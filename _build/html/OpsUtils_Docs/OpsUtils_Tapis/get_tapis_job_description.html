get_tapis_job_description#

get_tapis_job_description(t, tapisInput)

Generate a complete Tapis v3 job description from a concise input dict. The helper:

  1. Resolves input baseURL

    • If storage_system_baseURL is missing, it is derived from storage_system:

      • â€œmydataâ€ â†’ tapis://designsafe.storage.default/

      • â€œcommunityâ€ â†’ tapis://designsafe.storage.community

      • â€œpublishedâ€ â†’ tapis://designsafe.storage.published

    • The final sourceUrl used in fileInputs is: â€œ/<input_folder>â€.

  2. Ensures a concrete appVersion

    • If appVersion is missing or equals â€œlatestâ€, the function resolves a pinned version via OpsUtils.get_latest_app_version(t, ).

    • If it cannot resolve, the function returns -1 with a helpful message.

  3. Validates inputs

    • OpenSees-Express (appId == â€œopensees-expressâ€; defaults execSystemId to wma-exec-01 if not given): Required: [â€˜nameâ€™,â€™appIdâ€™,â€™appVersionâ€™,â€™maxMinutesâ€™,â€™archive_systemâ€™, â€˜storage_systemâ€™,â€™input_folderâ€™,â€™input_filenameâ€™]

    • HPC OpenSees apps (e.g., opensees-mp-s3, SP variants): Required: [â€˜nameâ€™,â€™appIdâ€™,â€™appVersionâ€™,â€™execSystemIdâ€™,â€™execSystemLogicalQueueâ€™, â€˜nodeCountâ€™,â€™coresPerNodeâ€™,â€™maxMinutesâ€™,â€™allocationâ€™,â€™archive_systemâ€™, â€˜storage_systemâ€™,â€™input_folderâ€™,â€™input_filenameâ€™]

  4. Assembles the job description

    • Express (Tcl)

      • parameterSet.envVariables = [{â€œkeyâ€:â€mainProgramâ€,â€valueâ€:â€OpenSeesâ€}, {â€œkeyâ€:â€tclScriptâ€,â€valueâ€: <input_filename>}]

      • fileInputs = [{â€œnameâ€:â€Input Directoryâ€,â€sourceUrlâ€: /<input_folder>}]

    • HPC (MP/SP)

      • Base job attributes: system, queue, resources, app id/version.

      • parameterSet.appArgs = [{â€œnameâ€:â€Main Scriptâ€,â€argâ€: <input_filename>}]

      • parameterSet.schedulerOptions = [{â€œnameâ€:â€TACC Allocationâ€,â€argâ€: fâ€-A {allocation}â€}]

      • fileInputs as above.

  5. Archive location

    • Express

      • archive_system == â€˜MyDataâ€™ â†’ designsafe.storage.default:${EffectiveUserId}/tapis-jobs-archive/${JobCreateDate}/${JobUUID}

      • archive_system == â€˜Tempâ€™ â†’ cloud.data:/tmp/${JobOwner}/tapis-jobs-archive/${JobCreateDate}/${JobName}-${JobUUID}

    • HPC (MP/SP)

      • archive_system == â€˜MyDataâ€™ â†’ same as above (MyData path)

      • archive_system == â€˜Workâ€™ â†’ ${WORK}/tapis-jobs-archive/${JobCreateDate}/${JobName}-${JobUUID} on the execution system

Parameters#

  • t (tapipy.tapis.Tapis) â€” Authenticated client (discovers username for MyData).

  • tapisInput (dict) â€” See required keys above; optionally include storage_system_baseURL to bypass auto-detection.

Returns#

  • dict â€” Ready-to-submit Tapis job description.

  • -1 â€” Validation failed, or baseURL/appVersion could not be determined.

Also prints missing keys (if any) and a confirmation message when inputs are complete.

Notes#

  • fileInputs uses the tapis:// scheme; the platform will stage inputs from that system/path.

  • For MyData auto-detection, the function queries your Tapis userinfo to insert your username.

  • The function resolves and pins appVersion when you omit it or set â€œlatestâ€.

  • Returns -1 if anything essential is missing; the console lists missing keys.

      - If `storage_system` contains "mydata": uses
        `tapis://designsafe.storage.default/<username>` where `<username>` is taken
        from `t.authenticator.get_userinfo().username`.
      - If `storage_system` contains "community": uses
        `tapis://designsafe.storage.community`
      - If `storage_system` contains "published": uses
        `tapis://designsafe.storage.published`
      - Else: prints a message and returns -1.

          archive_system == 'MyData' -> designsafe.storage.default under
              ${EffectiveUserId}/tapis-jobs-archive/${JobCreateDate}/${JobUUID}
